# ユーザーマニュアル

## はじめに

1. **ログイン** — 「GitHubでログイン」をタップして GitHub アカウントで認証
2. **リポジトリ選択** — 設定タブでリポジトリをタップして選択（最大5個）
3. **デフォルト設定** — 星アイコンをタップしてデフォルトリポジトリを設定

## カンバンタブ

GitInflow は Issue を4つのステータスタブで管理します:

| タブ | 説明 | ラベル |
|-----|------|-------|
| **Backlog** | ステータスラベルなしの Issue | (なし) |
| **Ready** | 作業準備ができた Issue | `status:ready` |
| **In Progress** | 作業中の Issue | `status:inProgress` |
| **Done** | 完了した Issue（自動クローズ） | `status:done` |

## ステータス変更

### スワイプ操作
- **右スワイプ** → 次のステータスに進める（例: Backlog → Ready）
- **左スワイプ** → 前のステータスに戻す（例: Ready → Backlog）

### コンテキストメニュー
- Issue を**長押し**すると全ステータスオプションが表示されます

### Issue 詳細
- Issue をタップすると詳細、コメント、ステータス変更ボタンが表示されます

## Issue 作成

- Backlog タブの **+** をタップ
- 投入先リポジトリを選択
- タイトルと本文を入力

## 共有シート

他のアプリから共有して Issue を作成:
1. 任意のアプリで共有ボタンをタップ
2. 「GitInflow Share」を選択
3. リポジトリとモード（新規作成 or 既存に追記）を選択

## ウィジェット

ホーム画面に GitInflow ウィジェットを追加:
1. ホーム画面を長押し
2. + をタップしてウィジェットを追加
3. 「GitInflow」を検索
4. Small（In Progress 件数）または Medium（全ステータス件数）を選択

## Pull Requests

PRs タブには選択リポジトリのオープンな Pull Request が表示されます。
