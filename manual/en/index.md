# User Manual

## Getting Started

1. **Login** — Tap "GitHubでログイン" to authenticate with your GitHub account
2. **Select Repositories** — Go to Settings tab, tap repositories to select (up to 5)
3. **Set Default** — Tap the star icon to set a default repository for new issues

## Kanban Tabs

GitInflow organizes your issues into 4 status tabs:

| Tab | Description | Label |
|-----|-------------|-------|
| **Backlog** | Issues without a status label | (none) |
| **Ready** | Issues ready to be worked on | `status:ready` |
| **In Progress** | Issues currently being worked on | `status:inProgress` |
| **Done** | Completed issues (auto-closed) | `status:done` |

## Changing Status

### Swipe Actions
- **Swipe right** → Move to next status (e.g., Backlog → Ready)
- **Swipe left** → Move to previous status (e.g., Ready → Backlog)

### Context Menu
- **Long press** an issue to see all status options

### Issue Detail
- Tap an issue to see details, comments, and status change buttons

## Creating Issues

- Tap **+** in the Backlog tab
- Select the target repository
- Enter title and body

## Share Extension

Share content from any app to create an issue:
1. Tap the Share button in any app
2. Select "GitInflow Share"
3. Choose a repository and mode (New Issue or Append to Existing)

## Widget

Add the GitInflow widget to your home screen:
1. Long press on the home screen
2. Tap + to add widgets
3. Search for "GitInflow"
4. Choose Small (In Progress count) or Medium (all status counts)

## Pull Requests

The PRs tab shows open pull requests from your selected repositories.
