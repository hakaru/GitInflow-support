# GitHub Issue でカンバン運用する方法

*2026年3月22日*

Jira や Trello がなくても、カンバンワークフローは回せます。コードが GitHub にあるなら、タスク管理も GitHub で。GitInflow を使ったセットアップ方法を紹介します。

## 4列のアプローチ

シンプルなカンバンボードは4列で構成されます：

| 列 | 意味 | いつ使う |
|---|------|---------|
| **Backlog** | 新しいアイデア、バグ報告、機能要望 | Issue が作成されたが未優先の段階 |
| **Ready** | トリアージ済み、着手可能 | やると決めたもの |
| **In Progress** | 現在作業中 | 手をつけたとき |
| **Done** | 完了 | 作業が終わったとき |

## GitInflow のマッピング

GitInflow は**ラベル**でステータスを表現します：

- ラベルなし → **Backlog**
- `status:ready` → **Ready**
- `status:inProgress` → **In Progress**
- `status:done` → **Done**（Issue も自動クローズ）

ポイントは、GitHub にカンバン列の機能がなくても、ラベルで同じことが実現できるということ。ラベルは GitHub API の一部なので、どのツールからでも読み書きできます。

## 日常のワークフロー

### 朝：Backlog のトリアージ

1. GitInflow を開く → Backlog タブ
2. 夜のうちに届いた新しい Issue をスキャン
3. 重要なものを右スワイプで **Ready** に移動
4. ラベルやマイルストーンをつける

### 日中：進捗の追跡

1. **Ready** から Issue を選んで **In Progress** にスワイプ
2. 作業する
3. スワイプアクションでメモ的にコメントを残す
4. 終わったら **Done** にスワイプ — Issue が自動クローズ

### 夕方：振り返り

1. **Done** タブ — 今日何を完了したか
2. **In Progress** — 滞っているものはないか
3. ホーム画面のウィジェットで件数をチラ見

## マルチリポジトリのコツ

複数リポジトリ（個人用、仕事用、サイドプロジェクト）を使う場合：

- 設定で**最大5つ**を選択
- **フィルタチップ**で1つのリポジトリに集中
- **デフォルトリポジトリ**を設定すると Issue 作成が高速化
- Share Extension もデフォルトリポジトリを使う

## なぜ GitHub Projects ではなくラベルなのか

GitHub Projects はチーム向けには優秀ですが、個人利用には重い：

- ラベルは**ユニバーサル** — どの GitHub ツールでも動く
- ラベルは**シンプル** — ボードの設定が不要
- ラベルは**ポータブル** — アプリを変えてもデータはそのまま
- GitInflow は初回使用時にステータスラベルを自動作成

## はじめよう

1. GitInflow をインストール
2. リポジトリを選択
3. Backlog のトリアージを開始
4. 習慣にする：Backlog → Ready → In Progress → Done

セットアップは30秒。整理されていない Issue には二度と戻れなくなります。
